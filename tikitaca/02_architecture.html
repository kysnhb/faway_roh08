Chapter 02

앱 구조 — WebView와 CDN

이 장은 리포트에서 가장 중요한 장이다. 나머지 모든 장의 유효 범위를 여기서 정의한다.

실행 구조

MainActivity (com.megagiant.tikitaca.one)
  └ mWebView  ←  https://cdn.megagiant.co.kr/ckwykr/onestore.html?time=<ts>
       └ MainActivity$5.shouldInterceptRequest()
            → js/json/png/mp3/html 요청을 번들 assets/assets/ (182MB)에서 우선 제공
            → 없으면 CDN에서 다운로드
항목값판정
실행 형태Android WebView 웹빌드확정 org.cocos2dx 0건 / WebView 54회
게임 코드 원본https://cdn.megagiant.co.kr/ckwykr/onestore.html확정 dex 상수문자열
번들 assets 역할로컬 캐시/프리로드 ("Load file from res")확정
네이티브 코드량자체 클래스 56개, 약 2,000줄 shim확정
유일한 .solibdatastore_shared_counter.so = AndroidX DataStore 표준 헬퍼확정 게임로직·안티치트 없음
JS↔네이티브 브리지WebViewCompat.WebMessageListener (name=OneStore)확정 @JavascriptInterface보다 안전한 선택
함의 3가지
  1. 분석한 index.dc249.js는 출고 시점 스냅샷이다. 라이브 버전은 CDN에서 갱신된다 → 스토어 심사 없이 게임 로직·밸런스·상점을 바꿀 수 있다.
  2. 게임플레이·경제·상점 로직의 100%가 JavaScript다. 네이티브 보호막이 전혀 없다.
  3. 보안 모델은 전적으로 서버 권한에 의존한다. 클라이언트 측 방어는 사실상 0이다.
