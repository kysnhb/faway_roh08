가장 큰 한계 — 왜 밸런스 수치가 없나

이 게임은 Unity(IL2CPP/Mono)처럼 enum·테이블로 설계가 드러나는 구조가 아니다. cocos2d-x C++ 컴파일 바이너리라, .dynsym은 함수 이름(시그니처)만 주고 본문(로직·상수)은 없다. 골드 획득량·강화 비용·드랍률·스탯 같은 수치는 bzStateGame 함수 안에 있어, Ghidra/IDA로 네이티브 디컴파일을 해야 확정된다. 이번 정적 분석 범위 밖이다.

내가 판단한 것 (질문 없이 결정)

  • 대상 = 직전에 분석한 Anger of Stick 5, 출력 = D:\newgaems\angerofstick5\ (참조 구현과 동일 루트)
  • 장르 = action (economy 게임 아님). 플레이북 Unity 경로를 cocos2d-x C++ 경로로 적응
  • 팔레트 = 아이콘 추출 틸블루+그린. 폰트 = 게임이 Arial 사용 → 독점폰트 임베드 대신 시스템 Arial 스택(라이선스 안전)
  • 마스코트 = 캐릭터가 부위 조립식이라 온전한 스프라이트 부재 → 아이콘 스틱맨 사용
  • 스테이지 60 = 업적 Level5~60에서 역추론(추정 표기)

검증하지 못한 것

리포트의 스크롤 진행바·마스코트 애니메이션은 실제 브라우저에서 최종 확인이 필요하다(정적 생성 환경에서 스크롤 이벤트 검증 불가). 레이아웃·링크·이미지·가로 스크롤 없음은 자동 검증했다.
